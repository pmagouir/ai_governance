# Pilot Task 6: The Delegation Map

## Overview

**Week:** 5-6 (April 30) | **Competency:** Delegation (Strategic) | **Modality:** All three

This task elevates delegation from individual decisions (Task 3: "which skill for this scenario?") to strategic planning: "which of my unit's recurring tasks should be delegated to AI, in what order, and with what governance safeguards?" This is the bridge to the capstone, where participants design teachable workflows for their units.

## Stage Detection

- **First contact or "start Task 6"** → Stage 1: INTRODUCE
- **Submits delegation map** → Stage 3: ASSESS
- **Submits revision** → Stage 3b: REASSESS
- **"Package this" or "I'm done"** → Stage 5: PACKAGE

## Stage 1: INTRODUCE

### Task: The Delegation Map — Strategic AI Deployment Plan

**What you're building:** A strategic map of your unit's 10 most recurring tasks, classified on a 2x2 matrix, with your top 3 deployment targets identified and fully specified with modality assignments and governance tiers.

**Why this matters:** Individual delegation judgment (Task 3) answers "can I delegate this task right now?" Strategic delegation (this task) answers "how should my unit systematically deploy AI across our recurring work?" This is the competency that determines whether AI adoption at DC CAP is ad hoc or planned. Your delegation map becomes the foundation for the teaching module in your capstone.

**The 2x2 Delegation Matrix:**

```
                        AI-Ready
                           ↑
              ┌────────────┬────────────┐
              │            │            │
              │  EXPLORE   │ DEPLOY NOW │ ← High ROI
              │            │            │
              │  Low freq  │  High freq │
              │  AI-ready  │  AI-ready  │
              │            │            │
    Low Freq ─┼────────────┼────────────┼─ High Freq
              │            │            │
              │  LEAVE IT  │ HUMAN CORE │
              │            │            │
              │  Low freq  │  High freq │
              │  Human-req │  Human-req │
              │            │            │
              └────────────┴────────────┘
                           ↓
                    Human-Required
```

| Quadrant | Meaning | Action |
|----------|---------|--------|
| **Deploy Now** | High frequency + AI-ready | Automate or augment immediately. Highest ROI. |
| **Human Core** | High frequency + Human-required | Frequent and judgment-dependent. Protect this time. AI can assist at the margins but the human is essential. |
| **Explore** | Low frequency + AI-ready | Build a template or skill for when it recurs. Not urgent but worth systematizing. |
| **Leave It** | Low frequency + Human-required | Rare and judgment-heavy. AI adds minimal value. Don't invest time here. |

**Time estimate:** 90-120 minutes for the full delegation map (10 tasks + 3 deployment targets), plus 30-45 minutes for revision.

**Prerequisite:** Tasks 1-5 should be completed. This task builds directly on your delegation judgment from Task 3 and applies it at the unit strategy level.

**The AI-Readiness Test:**

For each task, answer these three questions to determine AI-readiness:

1. **Repeatable structure?** Does this task follow a consistent template or pattern each time, or does every instance require a unique approach?
2. **Describable inputs?** Can you describe what Claude needs to know in a prompt, or does the task require real-time human context that's hard to articulate (e.g., reading a room, interpreting body language, navigating a live conversation)?
3. **Recoverable errors?** If Claude gets the output wrong, can you catch and fix it before harm occurs? Or would an error be irreversible or high-consequence before you could intervene?

**Yes to all three → AI-ready (Deploy Now or Explore, depending on frequency)**
**No to one or more → Human-required (Human Core or Leave It, depending on frequency)**

Use these questions in your placement rationale. They give you a concrete framework for the AI-readiness axis.

---

**Your deliverable:**

**For all 10 tasks, provide:**
1. **Task description** — Specific enough to classify. Name the deliverable, the audience, and how often it happens. "Draft the monthly counselor update email (once/month, goes to 30+ counselors)" is specific. "Handle emails" is not.
2. **Quadrant placement** — Deploy Now, Human Core, Explore, or Leave It
3. **Placement rationale** — Why this task belongs in this quadrant. Reference both frequency evidence (how often it actually happens) and AI-readiness reasoning (what makes it suitable or unsuitable for AI).

**For your top 3 "Deploy Now" tasks, also provide:**
4. **Modality assignment** — Automation, Augmentation, or Agency. With rationale explaining the human-AI balance this task requires.
5. **Governance tier** — Which data tier applies? What handling requirements does this create for AI use?
6. **First delegation step** — What's the first concrete action to deploy AI on this task? Name the skill to use (if one exists) or describe the prompt template to build.

**Submit your complete map.**

## Stage 3: ASSESS

### Assessment Approach

This task is more judgment-based than Tasks 1-4 because there is no single correct answer for how to classify a unit's work. Different units have different task portfolios. The skill focuses on:

1. **Internal consistency** — Does the rationale support the placement?
2. **Specificity** — Are tasks described concretely enough to evaluate?
3. **Governance accuracy** — Are the tiers correct given the data involved?
4. **Strategic quality** — Are the deployment targets genuinely the highest-ROI choices?

### Rubric

**Dimension 1: Task Specificity**
- **Exceeds Standard** — All 10 tasks name a specific deliverable, a specific audience or recipient, and a frequency indicator (weekly, monthly, per-cycle, per-event). Each task is distinct and couldn't be confused with another.
- **Meets Standard** — 8-9 tasks are specific. 1-2 are somewhat vague but classifiable.
- **Approaching Standard** — 5-7 tasks are specific. Several are categories rather than tasks ("handle partner communications" instead of "draft monthly partner enrollment verification reminder").
- **Below Standard** — Most tasks are too vague to meaningfully classify. Multiple tasks overlap or could be the same task described differently.

*Assessment guidance:* The specificity test has three components. Name each one when scoring:
- **Deliverable:** Does the task name what's produced? ("Draft email" vs. "handle emails")
- **Audience:** Does the task name who receives it? ("to university admissions directors" vs. "to partners")
- **Frequency:** Does the task indicate how often? ("monthly" or "3x per cycle" vs. no frequency)

**Dimension 2: Quadrant Logic**
- **Exceeds Standard** — Every placement rationale references specific evidence for frequency (cites how often the task actually occurs) AND specific reasoning for AI-readiness (explains what makes AI suitable or unsuitable, using the 3-question test or equivalent reasoning). Quadrant placements are internally consistent with the rationale.
- **Meets Standard** — Most rationales address both axes. 1-3 may be stronger on one axis than the other. At least 7 of 10 rationales reference BOTH frequency evidence AND AI-readiness reasoning.
- **Approaching Standard** — Rationales mostly address one axis (usually frequency) but neglect the other (AI-readiness reasoning is absent or generic). 4-6 rationales reference both axes.
- **Below Standard** — Placements appear arbitrary. Rationales don't connect to the quadrant framework, or rationales contradict the placement. Fewer than 4 rationales reference both axes.

*Assessment guidance:* The most common error is placing a task correctly on frequency but ignoring AI-readiness. "This happens weekly so it's Deploy Now" skips the second axis entirely. A weekly task that requires real-time human judgment about a unique student situation might actually be Human Core. Check whether the rationale addresses BOTH axes.

Also check for placement inconsistencies: if two tasks with similar characteristics end up in different quadrants, the rationales should explain why. If they don't, flag the inconsistency.

**Bright line for Approaching/Meets boundary:** If at least 7 of 10 rationales reference BOTH frequency evidence AND AI-readiness reasoning (using the 3-question test or equivalent reasoning) → Meets. If 4-6 do → Approaching. If fewer than 4 → Below.

**Calibration anchors:**
- **Approaching looks like:** "This task happens weekly so I put it in Deploy Now." (Addresses frequency only, ignores AI-readiness axis entirely)
- **Meets looks like:** "This task happens 3x per scholarship cycle (high frequency). It follows a consistent email template each time (repeatable structure), the inputs are always the same data fields from Salesforce (describable inputs), and if Claude gets the tone wrong I can catch it before sending (recoverable errors). Deploy Now with Augmentation modality." (Both axes addressed with specific evidence)

**Dimension 3: Deployment Target Quality**
- **Exceeds Standard** — Top 3 tasks are genuinely the highest-ROI choices from the Deploy Now quadrant. Modality assignments match the human-AI balance each task requires (not defaulting everything to Augmentation). First delegation steps are concrete and actionable — they name a specific skill to use or a specific prompt template to build.
- **Meets Standard** — Top 3 are reasonable choices. Modality assignments are defensible. First steps are present but may lack full specificity.
- **Approaching Standard** — Top 3 include a questionable choice (e.g., a task from Explore rather than Deploy Now, or a low-frequency task). Or modality assignments don't match the task requirements (e.g., assigning Automation to a task that clearly needs human judgment iteration).
- **Below Standard** — Top 3 are poorly chosen. Modality assignments show no understanding of the modality distinctions.

*Assessment guidance:* Check the modality assignments against the participant's own rationale. If they said a task "requires checking every output against current student records" in their rationale but assigned Automation, there's a contradiction. Automation means Claude handles 80%+ with human review. A task requiring record-by-record verification is Augmentation at minimum.

**Dimension 4: Governance Integration**
- **Exceeds Standard** — Governance tiers are correct for all 3 deployment targets. The participant explains what data the task involves, why that data maps to the assigned tier, and what handling the tier requires for AI use. The participant applies the aggregation guidance (N<10 rule) where relevant.
- **Meets Standard** — Governance tiers are correct. Reasoning is present but less detailed.
- **Approaching Standard** — Governance tiers are assigned but reasoning is absent or incorrect for 1+ targets. Or the participant assigns tiers based on the task name rather than the actual data involved.
- **Below Standard** — No governance tiers, or tiers are wrong for most targets.

*Assessment guidance:* Cross-reference against the governance framework's data classification table:
- Tasks involving individual student data → Tier 1 (Restricted, PROHIBITED with AI)
- Tasks involving financial details, small-cell data, embargoed info → Tier 2 (Restricted access)
- Tasks involving organizational knowledge, de-identified aggregates, program documentation → Tier 3 (Open access)
- Tasks involving published materials, public data → Tier 4 (Unrestricted)

If a participant assigns Tier 3 to a task that involves student-specific information, flag it. This is a governance error.

## Stage 3b: REASSESS

Re-score with delta comparison. Common revision patterns for this task: making vague tasks more specific, strengthening the AI-readiness side of quadrant rationales, and correcting governance tier assignments.

### Readiness Thresholds

- **Most dimensions at Meets or Exceeds:** Proceed to packaging. Note remaining refinements as optional stretch goals.
- **Multiple dimensions at Approaching:** Offer one more targeted revision cycle. Focus on the 1-2 dimensions with the largest gap. Allow the participant to revise just the weakest section rather than the full submission.
- **Any dimension at Below Standard:** Re-introduce the relevant concept before asking for another revision. Walk through one example together, then revise.

### How to Generate the Reflection Prompt and Facilitator Notes

**Reflection Prompt:** Before closing the assessment, ask the participant the task-specific reflection prompt above. Record their response. The prompt asks them to contrast two specific task placements, using concrete examples to articulate the boundary between AI-ready and human-required in their unit context.

**Facilitator Notes:** After you've assessed the portfolio, synthesize two insights:
1. **Pattern:** What does this participant's delegation map reveal about their unit's work? Is there a common pattern — e.g., all high-volume administrative tasks in Deploy Now, all student-facing work in Human Core?
2. **Discussion Question:** Frame one question that uses this specific delegation map to teach the whole cohort. The question should help others reflect on their own classification decisions.
