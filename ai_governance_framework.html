Executive Summary

This framework establishes the governance structure, data classification standards, and operational protocols for DC CAP Scholars to adopt Anthropic's Claude Enterprise AI platform. The framework balances organizational efficiency gains with rigorous data protection.

Claude Enterprise provides the security architecture required for responsible adoption. Anthropic does not use Enterprise customer inputs or outputs to train its models. The platform includes single sign-on, audit logging, and configurable data retention.

This document recommends a phased adoption approach that begins with a leadership pilot cohort using low-sensitivity organizational data, then expands access and data integration as the team builds fluency with the technology.

1. Strategic Rationale for Enterprise AI Adoption

DC CAP Scholars operates a comprehensive student success model that combines financial aid, intensive coaching, and university partnerships to serve over 800 scholars annually. The organization's strategic plan prioritizes operational efficiency, data-driven decision making, and scaling impact.

Claude Enterprise addresses these needs with:

  • Projects: Persistent workspaces loaded with organizational documents that Claude can reference across conversations
  • Skills: Building skills ensures best practices are captured for replication and efficiency
  • Connectors: Direct integrations with Microsoft 365 and nonprofit-specific platforms
  • Custom Retention: Organization-controlled data storage policies with full audit logging
  • Nonprofit Pricing: 75% discount for qualifying 501(c)(3) organizations

2. Data Classification Framework

The following classification system governs what organizational data can be used with Claude Enterprise, under what conditions, and by whom.

Tier Description Examples Claude Access
Tier 1
Restricted		 Individual PII, FERPA-protected data, SSN, financial aid details Scholar names + academic records, SSNs, financial aid details PROHIBITED — Never upload to Claude in any form
Tier 2
Sensitive		 Internal financial records, embargoed data, staff performance Budget models, Investment Portfolio allocations, disbursement schedules RESTRICTED ACCESS — Role-specific with least access defaults
Tier 3
Internal		 Organizational knowledge, program documentation, draft strategic analyses Strategic plans, grant narratives, internal process docs OPEN ACCESS — All licensed users. Upload to shared Projects
Tier 4
Public		 Published materials, public-facing content Annual reports, website content, press releases UNRESTRICTED — All users
Aggregation Guidance: Aggregated outcomes (retention rates, graduation rates by cohort year) are Tier 4 when anonymized. If any individual can be identified, the data reverts to Tier 1.
FERPA Compliance: DC CAP operates under FERPA requirements for student educational records. Student data protected by FERPA is classified as Tier 1 (Restricted) and must never be uploaded to Claude Enterprise in any form.
Anthropic's Data Commitment: Anthropic confirms that Enterprise customer data is never used for model training. Your data is protected with 90-day default retention (configurable) and full audit logging.

3. Phased Adoption Plan

DC CAP will roll out Claude Enterprise access in three phases, progressively expanding reach and data access as organizational fluency increases.

1
Leadership Pilot

Months 1–2

Objective: Build leadership fluency and validate governance protocols with low-risk data

  • 8 seats: CEO, COO, CSAO, Director of Program Strategy, plus rotating leads
  • All tiers with role dependencies
  • Design project infrastructure
  • Identify skill needs and build knowledge base
  • Success metrics: User adoption rate, tasks completed, time savings
2
Team Expansion

Months 3–5

Objective: Extend access and begin data integration

  • Extend access to coaching and operations teams
  • Introduce Tier 2 data access for authorized users
  • Enable Microsoft 365 connector
  • Train-the-trainer approach
  • Reduce retention to 60 days
3
Full Integration

Months 6–12

Objective: Organization-wide deployment with mature governance

  • Deploy all staff with role-appropriate permissions
  • Evaluate additional connectors (Candid, Salesforce)
  • Integrate into SOPs for grant writing, program reporting
  • Quarterly governance review cycle
  • Continuous optimization and training

6. Acceptable Use Policy

Required Practices

  • Verify data tier classification before uploading
  • Use shared organizational Projects for team knowledge
  • Review Claude outputs for accuracy before external use
  • Report accidental Tier 1 data uploads immediately
  • Separate personal projects from DC CAP projects

Prohibited Uses

  • Uploading any Tier 1 (Restricted) data
  • Automated decisions about scholar eligibility without human review
  • Automated outreach, hiring, dismissal without human review
  • Sharing Claude content externally without staff review and attribution
Anthropic's Data Handling: Anthropic confirms that Enterprise customer data is never used for model training. Your conversations and data are protected end-to-end with Enterprise security controls.

7. Incident Response Protocol

If accidental data upload occurs, follow these steps immediately:

  1. Immediate Containment

    Delete the conversation containing the data. Remove any shared links or access.

  2. Notification

    Notify AI Governance Lead within 2 hours; CEO/COO within 4 hours. Provide context and data description.

  3. Assessment

    Review audit logs for scope. Determine if data was shared or processed further.

  4. Remediation

    Tighten access controls. Update training. Document lessons learned.

  5. Documentation

    Record incident in governance log with resolution and preventive measures.

9. Alignment with Existing Policies

This AI Governance Framework aligns with and complements DC CAP's existing governance infrastructure:

  • GEAR UP Data Policy: All GEAR UP data handling complies with GEAR UP requirements. Tier 1 classification ensures restricted data is never uploaded.
  • Salesforce Data Governance: Claude can reference Salesforce via connector with appropriate access controls. SFDC data classification mirrors Claude tier system.
  • Employee Handbook / Acceptable Use: Claude Enterprise acceptable use policy extends existing IT security policies. All staff complete required training.
  • Partner MOUs: University and funder partnerships governed by existing MOUs. Claude Enterprise use must comply with contractual data handling obligations.
