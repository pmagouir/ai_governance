                    PORTFOLIO FORMAT: BEFORE vs. AFTER
================================================================================

PROBLEM STATEMENT
================================================================================
Original 15-20 page portfolios were comprehensive but rarely read.
- Participants didn't see peer work (no peer learning)
- Facilitators couldn't use portfolios to run peer discussions
- Board/KPMG reviewers couldn't scan to find key insights
- Nobody celebrated learning because portfolios were buried

Goal: Make portfolios SHORT, CLEAR, and IMPACTFUL for everyone.


BEFORE: FULL PORTFOLIO (15-20 PAGES)
================================================================================

STRUCTURE:
  1. Task Overview
     - Success criteria
     - Competency dimensions
  
  2. Initial Submission
     - Full governance classification work (2-3 pages)
     - Complete CTCC prompts (1-2 pages)
     - Skill selection narrative (½ page)
     - Verbatim Claude output (1-3 pages)
     - Audit findings narrative (½ page)
     - Full edit documentation with all tags/rationales (1-2 pages)
     - Human judgment analysis (½ page)
  
  3. Assessment
     - Complete rubric feedback for each dimension (2-3 pages)
     - Coaching notes (½ page)
  
  4. Revised Submission
     - Full revised work (2-3 pages, if revision occurred)
  
  5. Growth Trajectory
     - Before/after comparison table (1 page)
     - Overall readiness summary (½ page)
  
  6. Participant Reflection
     - 3-4 paragraph reflection (1 page)
  
  7. Facilitator Notes
     - Pattern observation
     - Discussion question

TOTAL: 15-20 pages (varies by task; Tasks 2-3 are longest)

READ TIME: 15-20 minutes per portfolio
FILE SIZE: 14 KB
AUDIENCE: Assessors, archive, appeals

USE CASE: "I need to see everything this person did"
PEER SHARING: Almost never (too long)
FACILITATOR DISCUSSION: Requires selective excerpting


AFTER: PORTFOLIO SUMMARY (1-2 PAGES)
================================================================================

STRUCTURE:
  1. Header Block (3 lines)
     Task | Participant | Unit | Date
     Competencies
     Week | Final Assessment Level
  
  2. Performance Snapshot (½ page)
     Table: 4 dimensions × 3 columns
     Column 1: Dimension name
     Column 2: Final level (Exceeds/Meets/Approaching/Below)
     Column 3: 1-sentence key evidence
  
  3. Growth Story (½ page)
     3-5 sentence narrative:
     - What was the main learning gap?
     - What improved?
     - What's transferable?
     - [For Tasks 2+] How does this compare to prior task?
  
  4. Participant Reflection (¼ page)
     - Data-grounded prompt
     - 2-3 sentence response
  
  5. Facilitator Insight (¼ page)
     - Pattern observation (1 sentence)
     - Discussion question (1 sentence)
  
  6. Multi-Task Comparison (¼ page, Tasks 2+ only)
     - Trajectory from prior task(s)

TOTAL: 1-2 pages (Task 1 ~1 page, Task 4 ~2 pages)

READ TIME: 5 minutes per portfolio
FILE SIZE: 12 KB
AUDIENCE: Peers, facilitators, board, KPMG

USE CASE: "Tell me what this person learned and why it matters"
PEER SHARING: Built-in (with discussion question)
FACILITATOR DISCUSSION: Ready-to-use discussion prompt included

KEY DESIGN CHANGES
================================================================================

1. FRONT-LOADING IMPACT
   Before: Show everything, reader extracts meaning
   After:  Lead with what was achieved, explain how

2. EVIDENCE-BASED SNAPSHOTS
   Before: Full narrative for each dimension
   After:  Level + one specific example per dimension

3. NARRATIVE GROWTH
   Before: Assessment feedback + revision narrative (separate sections)
   After:  Integrated growth story (one paragraph)

4. PEER CENTEREDNESS
   Before: Designed for assessor documentation
   After:  Designed to enable peer discussion (includes discussion question)

5. VISUAL SCANABILITY
   Before: Dense paragraphs, hard to skim
   After:  Tables, short paragraphs, section headers

6. PRESERVATION OF FULL RECORD
   Before: Everything in one place
   After:  Summary + appendix approach (summary for sharing, full for record)

WHY THIS MATTERS
================================================================================

The portfolio summary format creates three new capabilities:

1. PEER LEARNING AT SCALE
   - 10 people learning from each other's mistakes and insights
   - Discussion questions facilitate peer teaching
   - Shared language and patterns across units

2. FACILITATOR EFFICIENCY
   - Same assessment work, better output
   - Summary IS the peer discussion material
   - No separate "write summary for peers" step

3. CELEBRATION + ACCOUNTABILITY
   - Participants see their work printed and celebrated
   - Board sees evidence of learning outcomes
   - KPMG sees rigor and progression
   - Everyone reads the summaries (before: nobody read the 20-page versions)

BEFORE (excerpt from 20-page portfolio):

"REVISED SUBMISSION - SCENARIO C: Board presentation with Ward-level persistence

Tier Assignment: Tier 2 (Sensitive)

Reasoning: While overall persistence rates (87%) are de-identified aggregates 
that could be Tier 3, the Ward-level breakdown includes one Ward with only 6 
scholars. This falls below the N<10 small-cell threshold in the aggregation 
guidance, which makes re-identification possible through cross-referencing with 
public data about school locations and DC CAP partner lists.

Combination Test: If combined with public information about which schools 
operate in that specific Ward and DC CAP's published partner school list, the 
6 scholars in that Ward could potentially be identified. The small cell size 
is the key trigger that escalates this from aggregated data to re-identifiable 
aggregate.

Governance Action: Restricted access. The slide could be shared if the 
small-cell Ward is suppressed or combined with an adjacent Ward to bring N 
above 10. In current form, requires de-identification verification before use 
with AI tools."

(This is 1 scenario out of 3; full portfolio has this for every dimension)
