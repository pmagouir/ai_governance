# Dimension Glossary: How Assessment Names Map Across the System

This one-page reference resolves the naming differences you'll see across design documents, skill files, and portfolio summaries. All three layers assess the same underlying competencies. The names change because each task asks you to apply that competency at a different level of complexity.

## The Core Framework: Anthropic's 4D Competencies

| 4D Competency | What It Means (Plain Language) |
|---|---|
| **Diligence** | Know what data you're handling and what rules apply before you start |
| **Description** | Write a clear, complete prompt that tells Claude exactly what you need |
| **Delegation** | Pick the right tool and the right level of AI involvement for the job |
| **Discernment** | Check Claude's output, fix what's wrong, and document your judgment |

## How Dimension Names Evolve Across Tasks

Each task assesses all four competencies, but the dimension names reflect increasing complexity:

| 4D Competency | Task 1 (Guided) | Task 2 (Supported) | Task 3 (Independent) | Capstone |
|---|---|---|---|---|
| **Diligence** | Governance Accuracy | Governance Integration | Governance + Deployment Integration | Governance Pathway (in Use Case) |
| **Description** | Prompt + Delegation Quality | Prompt + Delegation Quality | Independent Pipeline Execution | 4D Self-Assessment (Description) |
| **Delegation** | (combined with Description above) | (combined with Description above) | Strategic Mapping Quality | Teaching Module Completeness |
| **Discernment** | Output Audit Rigor + Human Judgment Depth | Audit + Edit Quality | Before/After Documentation | Innovation Proposal Quality |
| **System-Level** | — | Pipeline Coherence (NEW) | — | Use Case Quality |

## Why the Names Change

**Task 1** asks: "Can you classify data and write prompts correctly?" The dimensions measure **accuracy** because there's an answer key.

**Task 2** asks: "Can you apply the framework to your real work?" The dimensions measure **integration** because you're connecting governance to prompts to audits in a coherent pipeline.

**Task 3** asks: "Can you run the whole thing independently and think strategically?" The dimensions measure **independence and deployment thinking** because there's no scaffolding.

**Capstone** asks: "Can you synthesize what you learned, teach it, and propose what's next?" The dimensions measure **synthesis and organizational impact**.

## Portfolio Display Names

When writing portfolio summaries, use the **task-specific dimension name** (middle columns above), followed by the 4D competency in parentheses. Example:

> **Governance Integration (Diligence):** Meets Standard — Correctly classified Tier 2 data with explicit reference to small-cell aggregation requirements.

## Assessment Levels (Same Across All Tasks)

| Level | What It Means |
|---|---|
| **Exceeds Standard** | Sophisticated, specific, connected work that goes beyond requirements |
| **Meets Standard** | Correct, specific, and clearly reasoned. Ready to proceed. |
| **Approaching Standard** | On the right track but missing specificity, connections, or key elements |
| **Below Standard** | Missing, incorrect, or too vague to assess. Needs re-teaching. |

## Capstone Self-Assessment Scale (Different from Task Levels)

The capstone uses a separate self-assessment scale for the participant's own growth reflection:

| Self-Assessment Level | What It Means |
|---|---|
| **Still Exploring** | New to this competency; rely on checklists and step-by-step guidance |
| **Building Awareness** | Understand the concepts; need support to apply them consistently |
| **Building Confidence** | Apply the competency with moderate independence; occasional gaps |
| **Proficient** | Apply the competency independently and consistently in your work |
| **Leading Practice** | Apply at a high level and can teach or mentor others |

## Quick Reference for Facilitators

When assessing, always use the **task-specific dimension names** from the SKILL.md file for that task. When writing portfolio summaries, add the 4D competency in parentheses for cross-task readability. When discussing growth trajectories, use the 4D competency names to show how the same underlying skill develops across all four tasks.
