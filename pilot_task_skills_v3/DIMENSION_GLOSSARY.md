# Dimension Glossary: Task-Specific Assessment Dimensions and 4D Competency Mapping

This reference explains how assessment dimensions work across the restructured task architecture. Each task targets a different cognitive demand type, so the dimensions measure genuinely different skills — not renamed versions of the same pipeline check.

---

## 1. The Core Framework: Anthropic's 4D Competencies

These four competencies are the foundation. Every task develops all four, but each task puts one competency in the spotlight.

| 4D Competency | What It Means (Plain Language) |
|---|---|
| **Diligence** | Know what data you're handling and what rules apply before you start |
| **Description** | Write a clear, complete prompt that tells Claude exactly what you need |
| **Delegation** | Decide how much Claude handles vs. how much you handle, and why |
| **Discernment** | Check Claude's output, fix what's wrong, and document your judgment |

## 2. How Dimensions Are Different Across Tasks

Each task asks a fundamentally different question. Because the cognitive demand is different, the assessment dimensions measure genuinely different skills.

| Task | Dim 1 | Dim 2 | Dim 3 | Dim 4 |
|---|---|---|---|---|
| **1: CLASSIFY** | Classification Accuracy | Boundary Reasoning | Combination Awareness | Governance Operationalization |
| **2: CREATE** | Design Completeness | Execution Quality | Edit Layer Rigor | Plan-vs-Reality Insight |
| **3: CRITIQUE** | Error Detection Accuracy | Severity Calibration | Self-Audit Depth | Corrective Transfer |
| **4: STRATEGIZE** | Task Mapping Quality | ROI Reasoning | Implementation Feasibility | Cross-Unit Awareness |
| **Capstone** | Use Case Impact | Self-Assessment Calibration | Teaching Module Completeness | Demo/Proposal Quality + Growth Narrative Coherence |

---

## 3. Why the Dimensions Are Different (Not Just Renamed)

The old glossary showed renamed versions of the same four pipeline dimensions across tasks. That made it look like participants were doing the same thing at increasing difficulty. They're not. Each task targets a different cognitive demand type, so the dimensions measure different things.

**Task 1 asks:** "Can you identify governance boundaries and reason about edge cases?"
Dimensions measure **analytical categorization** — whether you can accurately classify, reason about boundaries between tiers, handle combination scenarios, and translate classifications into operational constraints.

**Task 2 asks:** "Can you design and execute an AI-assisted workflow?"
Dimensions measure **design thinking and metacognitive analysis** — whether you can plan a complete workflow, execute it with quality, rigorously edit AI output, and reflect honestly on the gap between your plan and what actually happened.

**Task 3 asks:** "Can you evaluate AI output, diagnose failures, and improve systems?"
Dimensions measure **evaluative judgment and corrective reasoning** — whether you can spot errors, calibrate their severity, audit your own assessment process, and turn critique into actionable improvements.

**Task 4 asks:** "Can you think strategically about AI deployment at the unit level?"
Dimensions measure **strategic analysis and organizational planning** — whether you can map tasks accurately, reason about return on investment, assess implementation feasibility, and account for cross-unit dependencies.

**Capstone asks:** "Can you synthesize, calibrate, and teach?"
Dimensions measure **synthesis and multiplication** — whether you can demonstrate real impact, honestly assess your own growth, build something others can follow, and articulate a compelling vision for what comes next.

## 4. 4D Competency Mapping (For KPMG Reporting and Growth Tracking)

Each task has one PRIMARY 4D competency — the one that receives the most direct assessment. The other competencies are present and supporting but are not the central focus.

| 4D Competency | PRIMARY Task | Supporting Appearances |
|---|---|---|
| **Diligence** | Task 1: CLASSIFY | Tasks 2-4 (governance classification is required in every pipeline); Capstone (governance pathway in use case) |
| **Description** | Task 2: CREATE | Task 3 (Prompt Improvement Memo); Capstone (teaching module design) |
| **Delegation** | Task 4: STRATEGIZE | Task 2 (modality selection — hand off, draft together, figure it out together); Capstone (deployment recommendations) |
| **Discernment** | Task 3: CRITIQUE | Task 2 (edit layer rigor); Capstone (self-assessment calibration) |

This mapping means:
- Growth in **Diligence** is primarily measured by Task 1 performance, with supporting evidence from how governance shows up in Tasks 2-4.
- Growth in **Description** is primarily measured by Task 2 performance, with supporting evidence from prompt improvement in Task 3 and teaching design in the Capstone.
- Growth in **Delegation** is primarily measured by Task 4 performance, with supporting evidence from modality selection in Task 2 and deployment planning in the Capstone.
- Growth in **Discernment** is primarily measured by Task 3 performance, with supporting evidence from the edit layer in Task 2 and self-assessment honesty in the Capstone.

## 5. Assessment Levels (Same Across All Tasks)

All four tasks use the same four-point scale for assessment.

| Level | What It Means |
|---|---|
| **Exceeds Standard** | Sophisticated, specific, connected work that goes beyond requirements |
| **Meets Standard** | Correct, specific, and clearly reasoned. Ready to proceed. |
| **Approaching Standard** | On the right track but missing specificity, connections, or key elements |
| **Below Standard** | Missing, incorrect, or too vague to assess. Needs re-teaching. |

---

## 6. Capstone Self-Assessment Scale (Different from Task Levels)

The capstone uses a separate self-assessment scale for the participant's own growth reflection. This is not a facilitator rating — it's the participant's honest calibration of where they are.

| Self-Assessment Level | What It Means |
|---|---|
| **Still Exploring** | New to this competency; rely on checklists and step-by-step guidance |
| **Building Awareness** | Understand the concepts; need support to apply them consistently |
| **Building Confidence** | Apply the competency with moderate independence; occasional gaps |
| **Proficient** | Apply the competency independently and consistently in your work |
| **Leading Practice** | Apply at a high level and can teach or mentor others |

---

## 7. Three Ways to Work with AI (Plain Language)

The pilot uses plain language instead of technical terms for the human-AI division of labor.

| Plain Language | Technical Term | What It Means |
|---|---|---|
| **Hand it off** | Automation | You give Claude clear instructions; Claude produces the whole thing; you verify the result |
| **Draft together** | Augmentation | Claude gives you a starting draft; you reshape it with your knowledge and judgment |
| **Figure it out together** | Agency | You and Claude go back and forth on a complex problem; you make the decisions |

Use the plain language with participants. The technical terms appear in parentheses in facilitator-facing sections for reference.

## 8. Portfolio Display Names

When writing portfolio summaries, use the **task-specific dimension name** followed by the **primary 4D competency in parentheses**. This gives readers both the specific skill measured and the broader competency it maps to.

Example:

> **Classification Accuracy (Diligence):** Meets Standard — Correctly identified Tier 2 data classification with explicit reference to small-cell aggregation requirements and articulated why partner names alone are Tier 1 but combined with award amounts becomes Tier 2.

> **Edit Layer Rigor (Discernment):** Exceeds Standard — Documented 4 edits with specific rationale tags, connected two edit patterns back to prompt improvements, and identified a governance gap Claude missed in the original output.

---

## 9. Quick Reference for Facilitators

**When assessing:** Use the task-specific dimension names from the SKILL.md file for that task. The dimension names in Section 2 above are your rubric headers.

**When writing portfolio summaries:** Add the 4D competency in parentheses after the dimension name (see Section 8 above). This makes portfolios readable across tasks.

**When discussing growth trajectories:** Use the 4D competency names to show how the same underlying capability develops through different cognitive demands across tasks. For example: "Your Discernment started with edit layer rigor in Task 2, became the central focus in Task 3's error detection and severity calibration, and matured into honest self-assessment calibration in the Capstone."
